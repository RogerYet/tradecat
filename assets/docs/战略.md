

交易猫是 ai 时代的数据基建，中心化计算分发基于行情数据的派生数据（首先是币安交易所的行情数据），用于组合成熟 repo 以实现完整的交易生态，作为 ai 的交易工具

分层规划
输入/获取层：币安数据获取，http，repo等方法
处理/计算层：信号生成服务
消费/使用层：ai使用，前端，订单管理

架构分层封装思路

软件构造的层级结构示意

原子操作/数据（最底层的数据操作和存取）
/
表达式/运算（复杂计算逻辑）
/
语句块/控制流（构成程序的执行逻辑）
/
函数/方法（提供功能模块的定义和复用）
/
模块/组件/类（面向模块的方式组织相关功能）
/
包/库（可重用的代码库或功能集合）
/
子系统/域（特定的功能区域或业务领域）
/
应用/服务（具体的应用程序或服务）
/
系统/产品（完整的产品或系统）

1. 复杂系统的本质是递进式降熵与抽象封装，任何宏大系统无法凭空构建，只能通过逐层屏蔽底层细节的复杂度，利用标准化接口向上交付确定性，从而突破人类认知负荷极限与物理资源的单点约束

2. 系统架构演化的底层规律是分形与递归组合，从最微观的原子级数据读写到最宏观的商业产品交付，其组织逻辑具有绝对的同构性，即下层基本单元通过结构化排列生成上层模块，上层模块随之降维成更高层级组合链路中的基础单元

3. 价值与意义在层级跃迁中呈现非线性的涌现效应，底层的数据与表达式仅提供纯粹的物理状态转移与机械算力，毫无业务属性可言，只有当系统跨越函数层级向子系统与服务演进时，物理机制才开始与人类的社会需求及商业目的发生强耦合，最终在最高产品层实现价值的系统级爆发

4. 系统的稳定性要求与迭代灵活性在垂直维度上呈现严格的反比分布律，越接近底层的原子操作与控制流越需要极致的固化与稳定，其轻微扰动便会引发全局雪崩与级联失效，而越靠近顶层的服务与产品级则必须维持极高的塑性，以适应外部环境与商业逻辑的剧烈突变

5. 封装边界与隔离墙是抵抗系统熵增趋于无序崩溃的唯一物理学机制，类、组件、包与子系统的割裂并非单纯为了代码复用，其核心动机是建立局部错误、认知过载与逻辑污染的隔离带，确保局部模块的重构、变异或死亡被锁死在特定层级，而不至于摧毁整个产品矩阵的生命周期

6. 约束与驱动力在系统中具有双向非对称的因果关联，顶层产品的商业意图自上而下施加组合命令与业务导向，决定底层算力的流向，而底层原子数据的数据结构与物理极限则自下而上构成了顶层建筑不可逾越的客观天花板

7. 认知焦点的强制降维法则决定了协同创造的可能，在任意特定层级进行操作时，必须将紧邻的下一层级视为绝对可靠的黑盒实体，忽略其内部生灭机制，同时将紧邻的上一层级视为宏观边界约束，任何试图直接穿透两个以上层级的越维干预，必将导致架构的逻辑坍塌与管理失控
