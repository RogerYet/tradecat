# 0002 - governance-constitution-and-gates

用“维度模型 + 系统宪法 + 强制门禁”把复杂度关进笼子里：让坏改动尽量早失败（本地/CI），让 Agent 在小上下文内也能安全演进系统。

## In Scope

- 建立长期治理的“维度模型”：明确哪些维度必须统一口径、哪些允许多样化（以契约隔离）。
- 固化宪法 v1 与分层契约 v1，并定义其演进方式（版本化/弃用期/回滚策略）。
- 设计并落地强制门禁（不引入未经验证第三方依赖为前提）：
  - 依赖边界检查（禁止跨层 import / 循环依赖）
  - 契约变更检查（破坏性变更必须版本化）
  - 幂等键声明检查（所有写入路径必须声明幂等键）
  - 时间语义检查（UTC + 周期对齐的最小对齐用例）
  - 可观测三指标门槛（延迟/缺口率/重复率必须可计算）

## Out of Scope

- 不重构现有业务实现细节（不做“大搬家”）。
- 不修改 `config/.env`（生产配置只读）与任何敏感数据文件。
- 不引入未经验证的第三方治理依赖（例如 import-linter 一类）——除非另起任务评审通过。

## 阅读与执行顺序（必须严格遵守）

1. `CONTEXT.md`：确认现状、约束、风险、假设是否成立
2. `PLAN.md`：确认方案选择与回滚协议
3. `ACCEPTANCE.md`：锁定验收口径（不要边做边改标准）
4. `TODO.md`：逐项执行（每步必须跑 Verify）
5. `STATUS.md`：实时记录证据与状态迁移
